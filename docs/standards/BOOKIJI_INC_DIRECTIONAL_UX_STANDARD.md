# Bookiji Inc Directional UX Standard

## Standard Name

Directional UX Standard

## Status

Soft Standard  
Recommended  
Not mandatory

## Objective

Define a soft UX standard for all Bookiji Inc products where apps guide users based on context instead of relying primarily on navigation menus.

This is a soft standard, not a hard rule.

## Principle

Products should guide the user toward the next best action based on their current situation.

Replace menu-first UX with context-driven UX.

## Core Pattern

All Bookiji Inc apps should prefer:

1. Context
2. Suggestion
3. Action
4. Navigation (secondary)

## Layout Pattern

### Top Section - Context

Show:

- Current state
- Today
- What matters now

### Middle Section - Suggested Action

Show:

- Next best action
- Recommendation
- One-click CTA

### Bottom Section - Navigation

Show:

- Secondary features
- Tools
- History

## Goals

- Reduce cognitive load
- Make apps feel intelligent
- Improve engagement
- Create Bookiji Inc UX identity

## Soft Rule

Navigation is secondary.

Context and suggested actions are primary.

## Examples

Kinetix:

> You are ready for an easy run

MyAssist:

> You have 3 important tasks

Bookiji:

> You have 2 availability gaps

## Applicability

This standard applies to:

- Kinetix
- MyAssist
- Bookiji
- Future Bookiji Inc products
